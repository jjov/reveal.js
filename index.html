FLUTTER PROJECT STRUCTURE

BACKGROUND

Before start writing any line of code we need to make a decision about the project structure we want to follow for our flutter apps regardless the project you are working on.

PROJECT COMPOSITION

Package Name Description
functional_package_name This will contain functional packages like common, account...

ui

 Widgets that are part of the Graphic User Interface

bloc

 Business LOgic Components used to provide data to the UI and manipulate the data model defined

services

 It would contain the specification of the services to consume in the app

model

 Here we will define our Plain Old Dart Objects (PODO) which will represent our data model. (No business logic should be included here)

OPTIONAL PROJECT COMPOSITION COMPLEMENT

Package Name Description
functional_package_name This will contain functional packages like common, account...

...

 Here we will have all packages mentioned before

i18n

 Internationalization configuration for the UIs that make part of this specific functional package

repository

 In the future this package can be use to story the database repository that can be required

dao

 Incase data access objects can be required. If there is an ORM that can be use we can make it part of the model package, so this is not going to be required.
